# ウォーキングルート生成アプリ (Walking Route Generator)

現在地周辺のランダムなウォーキングルートを生成する、オフライン対応のPWA（Progressive Web App）です。毎日同じルートを歩くのに飽きた方に向けて、指定した距離でぐるっと一周するようなループ状のルートを自動生成します。

## 機能

*   **距離指定**: 0.5kmから10kmの範囲で、歩きたい目標距離をスライダーで設定できます。
*   **ランダムルート生成**: Overpass APIを利用してOpenStreetMapから道路データを取得し、A*アルゴリズムを用いて毎回異なるルートを生成します。
*   **ワンウェイ・ループ**: 行きと帰りで極力異なる道を通るよう経路を調整し、円を描くようなルートを作ります。
*   **履歴機能**: 過去に生成したルートの履歴を保存（最大10件）し、できるだけ過去に通った道を避けるようにルートを提案します。
*   **Google Maps 連携**: 生成したルートのウェイポイントを含むGoogle Mapsを開き、ナビゲーションを利用できます。
*   **PWA対応**: Service Workerにより、アプリをインストールしてオフラインに近い環境でも素早く起動できます（ネットワークデータ取得時はオンラインが必要）。

## 技術スタック

*   **フロントエンド**: HTML, CSS, JavaScript (Vanilla JS)
*   **地図描画**: [Leaflet.js](https://leafletjs.com/) (CartoDB Dark Matter)
*   **ルーティング・地図データ**: [Overpass API](https://wiki.openstreetmap.org/wiki/Overpass_API) (OpenStreetMap)
*   **PWA**: `manifest.json`, Service Worker (`sw.js`)

## インストールと起動方法

Node.js環境があれば、ローカルサーバーを立ち上げてすぐにご利用いただけます。

1. リポジトリをクローンまたはダウンロードします。
2. ターミナルでプロジェクトディレクトリに移動します。
3. `serve` などを使用してローカルサーバーを起動します。

```bash
npx serve -p 3000
```

4. ブラウザで `http://localhost:3000` にアクセスします。
   *(※位置情報の取得許可を求められた場合は、許可してください)*

## ルート生成の仕組み

1.  **データ取得**: 指定された距離の約 0.5倍の半径（最大 10km）の範囲内にある「歩行可能な道路（residential, footway, path 等）」のデータをOverpass APIから取得します。
2.  **グラフ構築**: 取得した道路データ（ノードとウェイ）を交差点ごとのグラフ構造に変換します。
3.  **中間地点の策定**: スタート地点からランダムな方角へ、目標距離に応じた中間地点を決めます。
4.  **行きと帰りの経路探索**:
    *   **行き**: スタート地点から中間地点まで、適度なランダムコストを加えたA*アルゴリズムで探索します。
    *   **帰り**: 中間地点からスタート地点まで、「行きで使った道」に重いペナルティを与え、別ルートを通るように探索します。
5.  **最適化**: 上記のプロセスを複数回（最大8回）、異なる方角や距離スケールで試行し、最も目標距離に近く、かつ過去のルートと重複が少ないものを採用します。

## 注意事項

*   OpenStreetMapのデータに依存しているため、地図データが存在しない場所や私有地がルートに含まれる場合があります。実際のルートを歩く際は、現地の交通ルールや案内に従ってください。
*   ブラウザの設定や環境によっては、位置情報の取得が正確に行われない場合があります。
